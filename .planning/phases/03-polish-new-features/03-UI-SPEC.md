# Phase 3 — UI Design Contract

> Visual and interaction contract for the Phase 3 conversion + Creator surfaces.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope:** FAQ section (CONTENT-01), Testimonials + animated counter (CONTENT-02),
> "Share this simulation" affordance (CONTENT-04), 9:16 branded story card (EXPORT-01/02/03),
> example-prompt chip copy refresh (CONTENT-03).
>
> **Non-goal:** No brand redesign. Every surface reuses the dark neon-gradient system already
> established in `app/globals.css`, `app/page.tsx`, `components/ResultView.tsx`. Do NOT invent
> new color tokens, fonts, or a component library. This contract is additive within the existing
> aesthetic.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Tailwind v4 CSS-first `@theme` in `app/globals.css` — no shadcn) |
| Preset | not applicable |
| Component library | none (hand-rolled components; `motion/react` for animation) |
| Icon library | `lucide-react` (v0.469.0 — already installed) |
| Font | Display: **Instrument Serif** (`font-display`) · Body/UI: **Inter** (`font-sans`) · Meta/labels: **mono** (`font-mono`) |

**shadcn gate:** Skipped by design. The project has a complete, founder-approved custom design system in `@theme`. Introducing shadcn would fight the existing tokens and the "don't redesign the brand" scope constraint. All new components are composed from the existing token vocabulary below.

**Token vocabulary (use ONLY these — do not add tokens):**

- Surfaces: `bg` `bg-soft` `surface` `surface-hi`
- Borders: `border` `border-hi`
- Text: `fg` `fg-soft` `fg-mute`
- Accents: `violet` `violet-glow` `cyan` `cyan-glow` `magenta` `amber`
- Utilities: `gradient-text` `gradient-border` `glow-violet` `glow-cyan` `text-glow` `grid-bg` `noise` `animate-pulse-glow` `animate-float-slow` `animate-shimmer`

## Spacing Scale

The existing codebase uses Tailwind's default 4px-based scale. New surfaces MUST stay on it. Observed conventions from `page.tsx` / `ResultView.tsx`:

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Chip inner gaps, icon-to-text gaps |
| sm | 8px | Compact stacks (`gap-2`), chip flex-wrap gaps |
| md | 16px | Card gaps in a grid (`gap-4`), default inner rhythm |
| lg | 20–24px | Card padding (`p-5` / `p-6` / `p-7`), inter-item gaps |
| xl | 32px | Sub-section gaps within a landing block |
| 2xl | 48px | Grid margins-top before card rows (`mt-14` ≈ 56px is the established landing rhythm) |
| 3xl | 64px+ | Landing section vertical padding — **match existing** `py-20 md:py-28` |

Card corner radii (match existing, do not deviate):
- Landing section cards / testimonial cards: `rounded-2xl` or `rounded-3xl`
- Chips / pills / badges: `rounded-full`
- Buttons: `rounded-xl` (compact) / `rounded-2xl` (hero-scale CTA)

**Exceptions:**
- Story-card export is a fixed **1080×1920px** canvas measured in absolute px (see EXPORT section) — it is NOT bound to the Tailwind scale because it renders outside the DOM flow (server `ImageResponse` / client rasterization). Its internal spacing uses the 8-point grid at export scale (multiples of 8 up to 96).

## Typography

Landing + result surfaces (DOM). Sizes below map to the existing Tailwind classes already in use — reuse them, do not introduce new arbitrary sizes.

| Role | Size (class) | Font | Weight | Line Height |
|------|-------------|------|--------|-------------|
| Section eyebrow | 12px (`text-xs`) uppercase `tracking-[0.22em]` | mono | 400 | default |
| Body | 14–16px (`text-sm` / base) | sans | 400 | 1.5 (`leading-relaxed`) |
| Card heading | 20–24px (`text-xl` / `text-2xl`) | display | 400 | 1.2 (`leading-tight`) |
| Section headline | 36–48px (`text-4xl md:text-5xl`) | display | 400 | 1.1 (`leading-tight`) |
| Counter numerals | 40–72px (`text-4xl md:text-6xl`) | display | 400 | 1.0 |

**Weights:** The design system runs on exactly two weights — **regular (400)** for display serif and most body, **semibold (600)** for buttons/CTA labels and plan badges. Do not introduce a third weight. `font-display` (Instrument Serif) is used at its natural single weight for all large numerals and headings.

Story-card export typography is specified separately in the EXPORT section (different scale — must be legible at phone-overlay size).

## Color

The palette is fixed by `@theme`. 60/30/10 split as it already operates on the landing page:

| Role | Token / Value | Usage |
|------|--------------|-------|
| Dominant (60%) | `bg` `#07060d` + `bg-soft` `#0d0a1a` + the fixed body radial-gradient wash | Page background, section backdrops |
| Secondary (30%) | `surface` `#110d22` + `surface-hi` `#1a1432` (typically at 40–60% opacity, e.g. `bg-surface/40`) | Cards, FAQ rows, testimonial cards, chips |
| Accent (10%) | `violet` `#a855f7` / `violet-glow` `#c084fc` (primary) and `cyan` `#22d3ee` / `cyan-glow` `#67e8f9` (secondary accent) | See reserved-for list below |
| Destructive / warning | `magenta` `#ec4899` (errors, "worst case", risk) · `amber` `#fbbf24` (demo-mode / caution flags) | Error text, worst-case scenario ring, demo badge |

**Accent reserved for (explicit — do not accent everything):**
- Primary CTA buttons (violet→magenta gradient): "Download story card", "Upgrade to Creator", "Simulate"
- Active/hover borders on interactive cards (`hover:border-violet-glow/50`)
- The animated social-proof counter numerals (may use `gradient-text`)
- Plan badges and the "Most popular" pill
- FAQ open-state chevron / active row left-accent
- Section eyebrows (violet-glow OR cyan-glow, alternating per section rhythm already on the page)

**Do NOT** accent: FAQ body text, testimonial body copy, resting card borders (`border-border`), or the counter labels — those stay `fg-soft` / `fg-mute`.

**Color contrast (WCAG):** `fg` on `bg` and `fg-soft` on `surface` pass AA. `fg-mute` (`#7d7aa3`) is for large/de-emphasized meta only — never for body paragraphs or anything below 14px that must be read. The story card must clear AA at overlay scale (large text ≥24px, ratio ≥3:1) — verify white/`fg` text over the darkest gradient stop, not the lightest.

## Surface-by-Surface Contract

### 1. FAQ section (CONTENT-01) — landing page

**Placement:** New `<section id="faq">` inserted **between Pricing and Final CTA** in `app/page.tsx`. Objections resolve right after price is seen, before the last CTA push. Add `FAQ` to the `Nav` LINKS array (`/#faq`).

**Layout:** Accordion (expand/collapse), single-column, `max-w-3xl` centered. Accordion beats a static list here — it keeps the page scannable in a TikTok recording and lets the viewer tap the objection that matches them. Default state: **all collapsed** (first item MAY be open to signal interactivity).

**Structure per row:**
- Container: `rounded-2xl border border-border bg-surface/40`, `divide-y divide-border/60` between rows (or per-row cards with `space-y-3`).
- Trigger: full-width `button`, `flex items-center justify-between`, `px-5 py-4`, question in `font-display text-xl leading-tight`, right-aligned lucide `ChevronDown` (`h-5 w-5 text-fg-mute`) that rotates 180° on open (`transition-transform`).
- Panel: answer in `text-sm md:text-base leading-relaxed text-fg-soft`, `px-5 pb-5`.

**Interaction / animation:** Use `motion/react` `AnimatePresence` + height/opacity for the panel (`initial={{height:0, opacity:0}}` → `animate={{height:"auto", opacity:1}}`). Respect `prefers-reduced-motion` (the CSS already disables smooth scroll under it — the panel should snap open with no height animation when reduced motion is set). Trigger needs a visible focus ring: `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-violet-glow/60`. Set `aria-expanded` on the trigger and `aria-controls` → panel `id`.

**Required content (4 objections — plain language, English, oracle-adjacent tone):**
1. **Data privacy** — "Is my decision private?" → We store your email and a subscription reference, nothing else. Your decision text is sent to the model to generate the simulation and is not saved to a profile or used for training.
2. **Accuracy disclaimer** — "How accurate are the probabilities?" → WhatIf is a decision-thinking tool, not a fortune teller. The three futures and probabilities are AI projections to widen your perspective — the final call is always yours.
3. **Refund policy** — "Can I cancel or get a refund?" → Cancel anytime from your account in two clicks; you keep access until the period ends. No lock-in, no hidden tiers.
4. **GDPR contact** — "You're in the EU — who do I contact about my data?" → We're an EU-based operation and GDPR-compliant. Email data requests (access, deletion) to the contact address in the footer and we'll action them.

Tone: direct, reassuring, faintly branded ("the oracle keeps no grudges" register from the existing pricing footnote). No legalese walls. GDPR contact MUST reference a real mailto (reuse the footer contact address — currently `hello@whatif.app`).

### 2. Testimonials + animated counter (CONTENT-02) — landing page

**Placement:** New `<section id="proof">` inserted **between Demo and Pricing** (social proof warms the visitor right before the price). Add nothing to Nav (keep the nav to 4 items max — FAQ takes the new slot).

**Sub-block A — Counter strip (animates on scroll):**
- 2–3 stat items, `font-display` numerals at `text-5xl md:text-6xl`, label under in `text-xs uppercase tracking-[0.15em] text-fg-mute` (mirrors the hero COUNTERS treatment already in `page.tsx`).
- The headline number (e.g. decisions simulated) counts up from 0 → target when it scrolls into view, using `motion/react` `useInView` + an animated number. Duration ~1.2s, `easeOut`. Fires **once** (`{ once: true }`).
- Numerals MAY use `gradient-text` for the hero stat. Under `prefers-reduced-motion`, show the final number immediately (no count-up).
- This strip is the primary "reads well in a TikTok recording" element — keep it large, high-contrast (`fg` numerals), generous spacing.

**Sub-block B — Testimonial cards (3–5):**
- Grid: `grid gap-5 md:grid-cols-3` (3 cards) or a 2-row layout if 4–5. Each card: `rounded-2xl border border-border bg-surface/40 p-6` with `hover:border-border-hi` transition (matches the "How it works" step cards).
- Card contents, top→bottom: a short quote in `font-display text-xl leading-snug text-fg` (the quote is the hero of the card — big, serif, high contrast); attribution row with a small gradient avatar dot (`h-8 w-8 rounded-full bg-gradient-to-br from-violet via-magenta to-cyan`) + name in `text-sm text-fg` + role/location in `text-xs text-fg-mute`.
- Optional top accent hairline `absolute inset-x-0 top-0 h-px bg-gradient-to-r from-violet to-cyan` (reuse ScenarioCard pattern) for visual continuity.

**Scroll-reveal animation:** Cards stagger in with `motion/react`: `initial={{opacity:0, y:20}}` → `whileInView={{opacity:1, y:0}}`, `viewport={{ once:true, margin:"-80px" }}`, `transition={{ duration:0.5, delay: i*0.1 }}` — exactly the cadence already used in `ResultView` ScenarioCard, for consistency.

**Content honesty:** These are marketing testimonials for a pre-launch product. Copy tone must be believable and specific ("I almost signed the lease — WhatIf showed me the 18-month version I hadn't pictured"), avatar dots are abstract gradients (not fake photos), and locations reuse the hero trust-strip cities (Berlin, Lisbon, NYC, London, Tokyo). Do not fabricate photos or full names implying real endorsements.

**Responsive:** Counter strip wraps to a centered column on mobile (`flex-wrap justify-center gap-x-10 gap-y-6`). Cards stack single-column under `md`.

### 3. "Share this simulation" affordance (CONTENT-04) — /result

**Location:** Inside `components/ResultView.tsx`, in the existing bottom action grid (the block currently holding "Ask another question" + the soft-paywall upsell). The share affordance is a **new tile in that grid**, gated by `me.plan` via the existing `useMe()` hook.

**Three states (drive off `me` — never flash the wrong one; `me === undefined` renders nothing, matching the existing `showUpsell` pattern):**

- **Creator (`me.plan === "creator"`):** Active tile. Primary action button "**Download story card**" (violet→magenta gradient, `rounded-xl`, lucide `Download` icon `h-4 w-4`). Secondary ghost action "Copy link" (`border-border-hi bg-bg/40`) if a share URL exists. Tile heading `font-display text-xl`: "Turn this into content." Sub: "Export a 9:16 card for TikTok, Reels, or Stories."
- **Free / Pro / anon (not Creator):** Upsell tile. Same visual slot, but the button reads "**Unlock Creator — €9/mo**" and routes through `UpgradeButton plan="creator"`. Heading: "Want to share this?" Sub: "Creator turns any simulation into a branded 9:16 card, built for the For You page." Include a small "Compare plans" ghost link to `/#pricing` (mirror the existing Pro upsell's secondary link).
- The **existing Pro upsell tile stays** for free users. For a **Pro** user, the Pro upsell disappears (they're a subscriber) but the **Creator share-upsell remains** — Pro→Creator is the live upgrade path this phase sells.

**Layout:** Extend the current grid. Recommended: `grid gap-4 md:grid-cols-3` where "Ask another question" is 1 col and the share/upsell tile spans `md:col-span-2` (matches the current upsell width). If a user is Creator (no Pro upsell needed), the grid becomes `md:grid-cols-2`.

**Interaction states for the Creator download button:**
- Idle → hover `brightness-110` (existing button pattern).
- Loading: disabled, spinner (reuse `animate-pulse-glow` dot or lucide `Loader2` with `animate-spin`), label "Rendering your card…".
- Success: brief inline confirmation "Saved — check your downloads" in `text-cyan-glow text-xs`, auto-clears after ~3s.
- **402 handling:** If a non-Creator ever reaches the export call (e.g. plan changed mid-session), the endpoint returns HTTP 402 with an upsell payload (EXPORT-03). The UI must surface it as the upsell tile / an inline notice — **never a silent failure or a raw error**. Notice copy: "Story-card export is a Creator feature." + the Creator CTA.

### 4. Branded 9:16 story card (EXPORT-01/02) — THE Creator differentiator

This is the shareable marketing artifact. It must look striking as a standalone image dropped into a TikTok/Reels editor. It renders **outside the normal responsive DOM** at a fixed canvas.

**Canvas:** exactly **1080 × 1920 px** (9:16). Output: **PNG**. All internal measurements below are in export-scale px on an 8-point grid.

**Rendering approach (executor guidance — pick per RESEARCH, but honor the font constraint):**
- Preferred: server route `app/api/export/route.ts` using Next's `ImageResponse` (`next/og`, built into Next 16) → deterministic, gate-able server-side (clean place to enforce the 402 for non-Creator), no client canvas quirks.
- **Font constraint (load-bearing):** there are **no local font files** in the repo (Inter + Instrument Serif load via CDN in the browser). `ImageResponse` needs the actual font binaries at render time — the export route MUST fetch/embed Instrument Serif and Inter (e.g. bundle the `.woff`/`.ttf` in `public/` or fetch from a font CDN and pass to `fonts:` in `ImageResponse`). Without this the card falls back to a system font and loses the brand. Add the two font files as part of this phase.
- Alternative (client rasterize via `html-to-image`/`dom-to-image`) is acceptable only if fonts are guaranteed loaded before capture; server rendering is preferred for the clean 402 gate.

**Visual layout (top → bottom), 1080×1920:**
1. **Background:** `#07060d` base with the brand radial gradient wash baked in — reproduce the body gradient: violet radial top-right (`rgba(168,85,247,0.22)`), cyan radial top-left (`rgba(34,211,238,0.14)`), magenta radial bottom (`rgba(236,72,153,0.12)`). Optional faint grid (`grid-bg` equivalent) at low opacity. This is what makes the card unmistakably WhatIf in a feed.
2. **Safe margins:** **96px** outer padding on all sides for content. Additionally reserve a **top ~220px band** and **bottom ~320px band** as low-density zones — TikTok overlays the caption/username at the bottom and UI controls at top-right. Keep critical text (question + recommendation) in the **vertical center 1080px**.
3. **Header (top band):** WhatIf wordmark — the gradient "?" tile (reuse Nav logo: `rounded-lg bg-gradient-to-br from-violet via-magenta to-cyan`) at ~72px + "What**If**" wordmark, "If" in `violet-glow`. Small mono eyebrow under it: `AI DECISION SIMULATION` in `fg-mute`, tracking wide.
4. **The question (center-upper):** the user's original decision, in **Instrument Serif** at ~**64–72px**, `leading-[1.1]`, `fg` (`#f5f3ff`), wrapped in curly quotes, max ~4 lines (truncate with … if longer). This is the emotional hook — largest text on the card.
5. **The three futures (center):** three compact rows — each `Best Case` / `Likely` / `Worst Case` with its probability. Reuse the ScenarioCard color language: Best = cyan→violet, Likely = violet→magenta, Worst = magenta→amber. Each row: tag chip (mono, ~24px) + a probability bar (full-width, ~12px tall, gradient fill to `pct%`) + the `pct%` numeral in `font-display` ~40px. Keep titles to one line each (truncate).
   - If space is tight, an acceptable alternative is a single **recommendation-forward** layout: the question + one bold recommendation line + a compact 3-dot probability legend. Executor picks based on typical content length; probabilities MUST appear either way (EXPORT-02).
6. **Recommendation strip (center-lower):** `THE CALL` mono eyebrow in `violet-glow`, then the recommendation in Instrument Serif ~**44px** `fg`, max 3 lines.
7. **Footer (bottom band, above the caption safe-zone):** small CTA line `whatif.app — simulate your decision` in `fg-soft` mono ~28px, centered. This is the growth loop — every reshared card is an ad.

**Legibility bar:** All text must be readable when the card is shown at phone scale inside a TikTok preview (roughly 1/3 physical size). That means: question ≥64px, no body text below ~28px, high contrast (light text on the dark base, not on the brightest gradient point), and probability bars thick enough (≥12px) to read as bars, not hairlines.

**Filename:** downloaded PNG named `whatif-simulation.png` (or slugged from the question). `Content-Disposition: attachment`.

**402 gate (EXPORT-03):** The export route returns **HTTP 402** with a JSON upsell payload `{ error: "creator_required", plan_needed: "creator", upgrade_url: "/#pricing" }` for free AND Pro users. The client maps this to the Creator upsell tile (surface 3). Never return an image or a silent 200 to a non-Creator.

## 6-Pillar Quality Guidance (for gsd-ui-checker)

1. **Copywriting** — Every new surface has finalized English copy above (FAQ answers, testimonial register, share/export CTAs, 402 notice, story-card footer). Error states name the problem + next step. No lorem, no placeholder.
2. **Visuals** — All new components reuse existing card / chip / gradient patterns (`rounded-2xl border border-border bg-surface/40`, gradient hairlines, glow shadows). Story card reproduces the brand gradient wash. No stray drop shadows or off-system radii.
3. **Color** — Only `@theme` tokens. 60/30/10 respected: dark surfaces dominant, `surface` cards secondary, violet/cyan accent restricted to the reserved-for list. Magenta/amber only for error/worst-case/caution. Card contrast passes AA; story card verified legible at overlay scale.
4. **Typography** — Two weights (400 / 600). Display serif for headings + numerals, Inter for body, mono for eyebrows. No new arbitrary sizes on landing surfaces. Story card: question ≥64px, no text <28px.
5. **Spacing** — 4px-based Tailwind scale throughout; landing sections match existing `py-20 md:py-28` rhythm and `mt-14` card-row offset. Story card on an 8-point grid with 96px safe margins + reserved TikTok overlay bands.
6. **Registry Safety** — No registries; nothing to vet. Font binaries for the export route are the only new asset — bundle from a trusted source (Google Fonts / the project's existing webfonts).

Additional accessibility checks the checker should confirm:
- FAQ accordion: keyboard operable, `aria-expanded` / `aria-controls`, visible focus ring, `prefers-reduced-motion` honored.
- Counter + card reveals: no motion under `prefers-reduced-motion` (final values shown immediately).
- Export button: disabled + labeled loading state; 402 never fails silently.

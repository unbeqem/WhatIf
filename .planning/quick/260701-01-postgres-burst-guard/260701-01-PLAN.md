# Quick Task 260701-01: Postgres-backed per-IP burst guard

## Why

Plan 01 specced Upstash Redis as the rate-limit backend for the ABUSE-01 per-IP
burst guard (5/min). The founder cannot reach the Upstash dashboard to provision
an account, so step 12 of the Phase 1 verify checklist was stuck as DEFERRED.

**Deviation from Plan 01:** switch the burst-guard backend from Upstash Redis to
Supabase Postgres. Rationale: every `/api/simulate` request already writes exactly
one `simulation_usage` row via `logUsage`, so a windowed `COUNT` by `ip_hash` is an
accurate request counter with no second datastore. Correct on Vercel serverless
(shared state, unlike an in-memory Map). One-file swap back to Redis later if needed.

## Tasks

### Task 1 — Rewrite `checkBurst` to query Postgres
- **files:** `lib/ratelimit.ts` (rewrite), `lib/upstash.ts` (delete — now unreferenced)
- **action:** Replace the Upstash `Ratelimit.slidingWindow` call with a Supabase
  admin `COUNT` of `simulation_usage` rows where `ip_hash = $ip` and
  `created_at >= now()-60s`. Block (`allowed:false`) when the prior count ≥ 5, so
  the 6th request in a rolling minute is rejected. Preserve the existing
  `BurstResult` shape and the fail-open contract (allow on demo mode / DB error).
- **verify:** `tsc` clean; `checkBurst` no longer imports `@/lib/upstash`.
- **done:** simulate route compiles and imports `checkBurst` unchanged.

### Task 2 — Index for the burst query
- **files:** `supabase/migrations/0002_burst_ip_index.sql` (new)
- **action:** Add `simulation_usage_ip_created_idx` on `(ip_hash, created_at desc)`
  so the windowed count is index-backed instead of a seq scan.
- **verify:** migration is valid SQL, idempotent (`if not exists`).
- **done:** founder applies via Supabase SQL editor / `supabase db push`.

## Verification (closes Phase 1 verify step 12)
Fire 6 rapid `POST /api/simulate` from one IP within 60s → the 6th returns HTTP 429
with body `{"error":"rate_limited",...}` (burst), distinct from the quota's
`limit_reached`. Confirms the Postgres burst guard engages.

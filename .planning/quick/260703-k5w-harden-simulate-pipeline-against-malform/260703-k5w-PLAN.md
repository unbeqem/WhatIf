# Quick Task 260703-k5w

## Problem

Bug report: on a free account, clicking Simulate "breaks — nothing comes back."
Investigation (prod DB, read-only): the API returns SUCCESS (free user got a
result at 12:13), real stored result shapes are clean, /result & /decision serve
200. So the happy path works. The unhandled gap: `lib/openai.ts` did
`JSON.parse(raw) as SimulationResult` with NO validation. If gpt-4o-mini ever
returns JSON without a `scenarios` array, the API returns 200 with a broken body
→ client stores it → `/result` `scenarios.map` throws → blank white page.

## Solution

- `lib/validate-simulation.ts`: shared `isValidSimulation` guard (server + client).
- `lib/openai.ts`: throw on non-JSON / invalid shape → clean retryable 500.
- `SimulateForm`: validate response before store/redirect; 40s AbortController
  timeout so the oracle never spins forever.
- `ResultView`: friendly "try again" card for an unrenderable stored result.
- `app/result/error.tsx`: route error boundary — no blank page on any throw.

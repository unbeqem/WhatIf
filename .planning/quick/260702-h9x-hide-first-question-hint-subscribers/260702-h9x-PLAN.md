# Quick Task 260702-h9x — Hide "first question" hint for subscribers

## Problem

The landing page's final CTA is "first question"-framed: the paragraph ends with
"The first one is free." and the button reads "Start your first simulation"
(`app/page.tsx:339-349`). For a Pro/Creator subscriber who has already run a
simulation this session, that framing is wrong and redundant.

## Solution

`app/page.tsx` is a server component, so mirror the existing `components/PricingCta.tsx`
pattern: extract the final-CTA paragraph + button into a small `"use client"`
component `components/FinalCta.tsx` that uses `useMe()` + `isSubscriberPlan()` and
reads `sessionStorage["whatif:last"]` to detect a prior question.

- Loading window / anon / free / no prior question → current copy unchanged
  ("The first one is free." + "Start your first simulation").
- Subscriber (pro/creator) **and** a question already asked → neutral copy
  ("Run another simulation", no "first one is free" line).

Graceful default (renders the free-framed copy while `me === undefined`) matches
PricingCta so there's no flash of the wrong state.

## Verification

- `npx tsc --noEmit` clean
- `npm run build` succeeds
- Anon/free landing still shows "Start your first simulation" + "The first one is free."

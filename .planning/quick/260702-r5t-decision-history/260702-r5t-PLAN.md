# Quick Task 260702-r5t — Decision history (Pro feature)

Delivers the "Save your decision history" Pro feature already advertised on the
pricing page but never built. All patterns reused from the existing codebase.

## Decisions (founder-confirmed)

- **Location:** dedicated `/history` page (+ nav link), not an /account section.
- **Re-view:** clicking an entry writes the stored result to
  `sessionStorage["whatif:last"]` and opens `/result` — reuses ResultView whole,
  no new detail UI.

## Changes

1. **Migration `supabase/migrations/0004_history.sql`** — `public.simulations`
   (`id uuid`, `user_id`, `input`, `result jsonb`, `summary`, `created_at`),
   index `(user_id, created_at desc)`, RLS `simulations_select_own`, writes
   service-role-only (no INSERT policy) — mirrors `simulation_usage` from 0001.
2. **`lib/db.types.ts`** — add `simulations` table Row/Insert/Update + a `Json`
   alias for the jsonb column.
3. **`lib/history.ts`** (new, server-only) — `saveSimulation()` (skips demo
   results; `summary = most_likely`, swallows errors like `logUsage`) and
   `listSimulations()` (newest first, cap 50).
4. **`app/api/simulate/route.ts`** — Step 4: after a successful run, if the
   actor is a pro/creator user, `await saveSimulation(...)`.
5. **`app/history/page.tsx`** (new, server) — auth-gate (→ /login), plan-gate
   (free → "History is a Pro feature" upsell), empty state, else list.
6. **`components/HistoryEntry.tsx`** (new, "use client") — card with question +
   summary + date; "View" → sessionStorage + `router.push("/result")`.
7. **`components/AuthNav.tsx`** — "History" link for pro/creator viewers.

## Verification

- `npx tsc --noEmit` clean
- `npm run build` succeeds; `/history` builds
- Demo-mode safe (no supabaseAdmin → save/list no-op, page shows gated/empty)
